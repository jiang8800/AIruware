# 项目结构通俗指南 (Project Structure Guide for Beginners)

欢迎！这个文档将用通俗易懂的方式（像盖房子一样）为你解释这个项目里各个文件的作用。

## 🏠 根目录：房子的地基和蓝图

根目录下的文件通常是项目的配置文件，就像房子的蓝图和施工规范。

-   **`package.json`** 📄 **(配方/购物清单)**
    -   这是最重要的文件之一。它告诉电脑这个项目叫什么，版本是多少，以及需要安装哪些“零件”（依赖包）才能运行。就像做菜的食谱，列出了所有需要的食材。
-   **`next.config.js`** ⚙️ **(装修规则)**
    -   这是 Next.js 框架的配置文件。它设定了网站运行的一些特殊规则，比如图片从哪里加载，或者链接怎么跳转。
-   **`tsconfig.json`** 📘 **(语法书)**
    -   这是 TypeScript 的配置文件。它告诉电脑我们用的是哪种“方言”来写代码，以及检查代码错误的严格程度。
-   **`.env` / `.env.local`** 🔑 **(保险箱)**
    -   这里存放敏感信息，比如 API 密钥、数据库密码等。这些东西不能公开给别人看，所以通常不会上传到 GitHub。
-   **`README.md`** 📖 **(说明书)**
    -   项目的介绍文档。通常包含项目是做什么的、怎么安装、怎么运行等信息。别人拿到你的项目，第一时间就是看这个。
-   **`.gitignore`** 🚫 **(黑名单)**
    -   告诉 Git（版本控制工具）哪些文件**不需要**上传到网上去。比如 `node_modules`（太大了，每个人自己安装就行）和 `.env`（太危险了）。

## 📂 文件夹：房子的各个房间

-   **`app`** 🚪 **(房间/页面)**
    -   这是存放网站页面的地方。
    -   `page.tsx`: 每一个文件夹里的 `page.tsx` 就代表一个网页。比如 `app/page.tsx` 是首页，`app/course/page.tsx` 就是课程页。
    -   `layout.tsx`: 这是一个“模版”或“骨架”。比如所有页面都有一样的导航栏和页脚，我们就把它们写在 `layout.tsx` 里，这样就不用每个页面都写一遍了。
    -   `globals.css`: 全局样式表，就像房子的整体配色方案（墙漆颜色、地板材质）。

-   **`components`** 🛋️ **(家具/组件)**
    -   这里存放可以重复使用的“积木”。
    -   比如 `Header.tsx` (导航栏)、`Footer.tsx` (页脚)、`CourseCard.tsx` (课程卡片)。
    -   你可以在不同的页面（房间）里摆放这些家具（组件），而不用每次都重新制作。

-   **`content`** 📚 **(图书馆/仓库)**
    -   这里存放你的教材内容。
    -   `chapters/*.md`: 你的每一章教材（比如 `unit-1.md`）都放在这里。网站会自动读取这些文件，把它们变成漂亮的网页展示出来。

-   **`lib`** 🛠️ **(工具箱)**
    -   这里存放一些辅助功能的代码。
    -   比如 `content.ts` 里可能有一个工具，专门用来“读取 markdown 文件”或者“计算学习进度”。它们不直接显示在页面上，但在背后默默干活。

-   **`public`** 🖼️ **(储藏室)**
    -   这里存放静态资源，比如图片、图标、字体等。
    -   放在这里的文件，可以通过网址直接访问。比如 `public/logo.png` 可以通过 `你的域名/logo.png` 看到。

-   **`scripts`** 🤖 **(自动化机器人)**
    -   这里存放一些自动执行的脚本。
    -   比如刚才写的 `migrate-content.js`，它就像一个机器人，帮你把一大本教材自动拆分成很多小章节。

## 总结

-   **想改页面内容？** 去 `app` 文件夹找对应的 `page.tsx`。
-   **想改教材文字？** 去 `content/chapters` 找对应的 `.md` 文件。
-   **想改导航栏或页脚？** 去 `components` 文件夹。
-   **想加图片？** 扔进 `public` 文件夹。

希望这个指南能帮你更好地理解你的项目！
